# Laravel API

REST API built with Laravel 13, token-based authentication via Sanctum, and OpenAPI documentation auto-generated by Scramble.

---

## Architecture

### Stack

| Layer | Technology |
|-------|-----------|
| Runtime | PHP 8.5 |
| Framework | Laravel 13 |
| Authentication | Laravel Sanctum (Bearer token) |
| Database | MySQL 8.4 |
| Cache / Queues | Redis |
| Email (dev) | Mailpit |
| API Docs | Dedoc Scramble (OpenAPI 3.1) |

### Docker Services (Sail)

| Service | Default Port |
|---------|-------------|
| Application (HTTP) | `80` |
| MySQL | `3306` |
| Redis | `6379` |
| Mailpit (SMTP) | `1025` |
| Mailpit (Dashboard) | `8025` |
| phpMyAdmin | `8080` |

### Authentication & Authorization

- Stateless authentication via **Laravel Sanctum** with Bearer tokens.
- Role-based access control using the `Role` enum (`Admin`, `Client`).
- Mandatory email verification (`MustVerifyEmail`).
- Password reset via email notification.

### API Endpoints

| Method | Endpoint | Auth | Description |
|--------|----------|:----:|-------------|
| `POST` | `/api/auth/register` | вЂ” | Register a new user |
| `POST` | `/api/auth/login` | вЂ” | Login, returns a Sanctum token |
| `POST` | `/api/auth/logout` | Bearer | Revoke the current token |
| `POST` | `/api/auth/forgot-password` | вЂ” | Send password reset email |
| `POST` | `/api/auth/reset-password` | вЂ” | Reset password via token |
| `GET` | `/api/email/verify/{id}/{hash}` | Signed URL | Verify user email |
| `POST` | `/api/email/resend-verification` | вЂ” | Resend verification email |
| `GET` | `/api/user` | Bearer | Return the authenticated user |

## Getting Started with Laravel Sail

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.

### Setup

```bash
# 1. Clone the repository
git clone <repository-url>
cd <project-folder>

# 2. Install PHP dependencies (no local PHP required)
docker run --rm \
    -u "$(id -u):$(id -g)" \
    -v "$(pwd):/var/www/html" \
    -w /var/www/html \
    laravelsail/php85-composer:latest \
    composer install --ignore-platform-reqs

# 3. Copy the environment file
cp .env.example .env

# 4. Start the containers
vendor/bin/sail up -d

# 5. Generate the application key
vendor/bin/sail artisan key:generate

# 6. Run database migrations
vendor/bin/sail artisan migrate

# 7. (Optional) Install Node dependencies and build assets
vendor/bin/sail npm install
vendor/bin/sail npm run build
```

The application will be available at `http://localhost`.

### Common Commands

```bash
# Start services
vendor/bin/sail up -d

# Stop services
vendor/bin/sail stop

# Run tests
vendor/bin/sail artisan test --compact

# Open in browser
vendor/bin/sail open
```

## OpenAPI Documentation

Interactive API documentation is auto-generated by [Scramble](https://scramble.dedoc.co/) directly from the source code.

Once the containers are running, access:

| Interface | URL |
|-----------|-----|
| Interactive UI | `http://localhost/docs/api` |
| OpenAPI spec (JSON) | `http://localhost/docs/api.json` |

> **Tip:** use the **Mailpit** dashboard at `http://localhost:8025` to inspect emails sent during development (account verification, password reset, etc.).
