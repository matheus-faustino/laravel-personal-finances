# Personal Finances API

REST API for personal finance management built with Laravel 13. Supports document upload (invoices/bills), AI-powered transaction extraction via **Gemini**, automatic categorization, and full CRUD for transactions and categories.

---

## Overview

Upload a PDF invoice and the API automatically:

1. Extracts all transactions using the **Gemini** AI model (`InvoiceTransactionExtractorAgent`).
2. Categorizes each transaction against your category list (`CategorizeTransactionsJob`).
3. Sets the document status through `uploaded → processing → processed` (or `failed` on error).

Everything runs asynchronously through Laravel's queue system.

## Getting Started with Laravel Sail

### Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed and running.
- A **Gemini API key** — obtain one at [Google AI Studio](https://aistudio.google.com/app/apikey).

### Setup

```bash
# 1. Clone the repository
git clone <repository-url>
cd <project-folder>

# 2. Install PHP dependencies (no local PHP required)
docker run --rm \
    -u "$(id -u):$(id -g)" \
    -v "$(pwd):/var/www/html" \
    -w /var/www/html \
    laravelsail/php85-composer:latest \
    composer install --ignore-platform-reqs

# 3. Copy the environment file
cp .env.example .env

# 4. Start the containers
vendor/bin/sail up -d

# 5. Generate the application key
vendor/bin/sail artisan key:generate

# 6. Run database migrations and seed initial data
vendor/bin/sail artisan migrate --seed
```

The application will be available at `http://localhost`.

### Gemini API Key

Open `.env` and set your key:

```dotenv
GEMINI_API_KEY=your_api_key_here
```

The AI agents use Gemini by default (`Lab::Gemini`). Without this key, document processing jobs will fail.

### Common Commands

```bash
# Start services
vendor/bin/sail up -d

# Stop services
vendor/bin/sail stop

# Run the queue worker (required for document processing)
vendor/bin/sail artisan queue:work

# Run tests
vendor/bin/sail artisan test --compact

# Open in browser
vendor/bin/sail open
```

## Document Processing Flow

### 1. Authenticate

```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password"
}
```

Copy the `token` from the response.

### 2. Upload a document (invoice PDF)

```http
POST /api/documents
Authorization: Bearer <token>
Content-Type: multipart/form-data

file=@invoice.pdf
name=My Invoice
```

The response will have `"status": "uploaded"` and the document `id`.

### 3. Start the queue worker

```bash
vendor/bin/sail artisan queue:work
```

The worker processes two chained jobs:

| Job | What it does |
|-----|-------------|
| `ProcessDocumentJob` | Sends the PDF to Gemini and creates transactions |
| `CategorizeTransactionsJob` | Assigns a category to each extracted transaction |

The document status updates to `processing`, then `processed` (or `failed` if an error occurs).

### 4. Check the result

```http
GET /api/documents/{id}
Authorization: Bearer <token>
```

```http
GET /api/transactions?document_id={id}
Authorization: Bearer <token>
```
